# BCI RC카 프로젝트

> **Muse 2, PyTorch, LSL, BLE(HM-10), Arduino**를 활용해  
> **뇌파(EEG) 기반으로 RC카를 실시간 제어**한 프로젝트입니다.

## 프로젝트 개요
이 프로젝트는 뇌파를 입력 인터페이스로 활용해, 사용자의 의도를 실시간으로 분류하고  
그 결과를 RC카 제어 명령으로 연결하는 **BCI(Brain-Computer Interface)** 시스템을 구현하는 것을 목표로 합니다.

최종 시스템은 다음 흐름으로 동작합니다.

1. **Muse 2** 헤드셋으로 EEG 신호를 측정
2. 주파수 대역별 특징(feature) 추출
3. **PyTorch 분류 모델**로 사용자 의도 분류
4. 분류 결과를 **BLE(HM-10)** 로 Arduino에 전송
5. Arduino가 RC카를 실제로 제어

이 저장소는 프로젝트의 **최종 결과물 중심 공개용 저장소**입니다.  
최종 발표 자료, 최종 실행 코드, 최종 모델을 포함합니다.

---

## 시스템 동작 정의

### 분류 클래스
- `left`
- `right`
- `blink`
- `none`

### RC카 제어 매핑
- `left` → 좌회전
- `right` → 우회전
- `blink` → 직진
- `none` → 직진

### 최종 제어 방식
최종 구현에서는 `blink`와 `none`을 모두 **전진 계열 상태**로 사용하고,  
`left`, `right`는 **직각 회전** 동작으로 연결했습니다.

## 기술 구성

### 하드웨어
- **Muse 2** EEG 헤드셋
- **HM-10 BLE 모듈**
- **Arduino 기반 RC카**

### 소프트웨어
- **PyTorch**: 뇌파 분류 모델
- **LSL (Lab Streaming Layer)**: 실시간 EEG 스트리밍
- **Python**: 실시간 추론 및 BLE 통신
- **Arduino**: RC카 모터 제어

---

## 특징 추출 방식
최종 파이프라인에서는 **4개 채널**과 **5개 주파수 대역**을 사용해  
총 **20차원 feature vector**를 구성합니다.

### 사용 채널
- TP9
- AF7
- AF8
- TP10

### 사용 주파수 대역
- Alpha: 8–12 Hz
- Beta 1: 12–20 Hz
- Beta 2: 20–30 Hz
- Gamma 1: 30–35 Hz
- Gamma 2: 35–40 Hz

즉,

- **4채널 × 5대역 = 20차원 feature**

로 구성됩니다.

## 프로젝트 의의
이 프로젝트의 의미는 단순히 EEG를 분류했다는 데 있지 않습니다. 초기에는 OpenViBE 기반으로 시스템을 구성했지만, 원하는 방식으로 세부 수정과 반복 실험을 진행하기 어렵다는 한계가 있었습니다. 그래서 기존 흐름을 **PyTorch 기반 파이프라인으로 직접 재구현**했고, 이를 통해 모델 구조와 처리 과정을 직접 수정하며 실험할 수 있는 환경을 확보했습니다.

이후 EEG 추출 과정에서는 **눈 깜빡임(blink)** 이 매우 강한 노이즈로 작용한다는 문제를 확인했습니다. 원래 `none`은 아무 생각을 하지 않는 기본 상태로서 직진 계열 동작과 연결하려 했지만, blink 신호를 함께 처리하면 오히려 제어 안정성이 떨어질 수 있었습니다. 따라서 `blink`를 별도 클래스로 분리하여 예외 처리함으로써 실제 제어에 더 적합한 구조로 정리했습니다.

또한 EEG는 사람마다 개인차가 매우 크고, 다양한 참여자의 데이터를 충분히 수집하는 데에도 현실적인 제약이 있었습니다. 이 때문에 범용 분류기보다는 **개인화된 BCI 제어**를 목표로 방향을 전환했고, 최종적으로는 한 사람의 데이터 중심 학습으로 정리했습니다.

마지막으로 실제 시연 단계에서는 **오분류**가 가장 큰 문제였습니다. 이를 줄이기 위해 단일 추론 결과만으로 바로 RC카를 움직이지 않고, **같은 라벨이 3회 연속 판별될 때만 명령을 전송**하도록 안정화 로직을 적용했습니다. 이 방식은 반응 지연을 증가시켰지만, 그 대신 의도하지 않은 동작을 줄여 RC카가 실제로 의도한 방향으로 움직이도록 만들었습니다.

결과적으로 이 프로젝트는 BCI가 단순한 오프라인 분류 실험을 넘어, **실제 물리 시스템(RC카) 제어로 이어질 수 있음을 보여주는 end-to-end 구현 사례**입니다. 동시에 노이즈가 크고 개인차가 심한 생체신호를 현실적인 제어 시스템으로 연결하기 위해, 어떤 설계 수정·예외 처리·안정화 전략이 필요한지를 구체적인 트러블슈팅 과정으로 보여준다는 점에서 의미가 있습니다.
